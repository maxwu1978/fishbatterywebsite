## 必须明确的边界

以下功能 `不要误判为已上线完整能力`：

### 1. Live chat 未启用

- `chat-config.js` 里当前 `crispWebsiteId` 为空
- `support-chat.js` 因此不会真正加载 Crisp
- 结论：
  - 站点当前 `没有正式 live chat widget`
  - 现在能用的是 `表单咨询`，不是 `真人即时聊天`

### 2. `/api/chat` 不是线上可用能力

- 本地开发有：
  - `server/support_server.py`
  - `POST /api/chat`
- 但这是本地 Python 服务能力，不是当前 Vercel 线上能力
- 当前生产环境下：
  - `https://www.getreelmate.com/api/chat` 返回 `404`
- 结论：
  - `contact.html` 里的即时支持台更像预留/演示结构
  - 不应再把它当成生产已可用 AI 客服

### 3. 正式支付未开启

- 当前测试支付、webhook、订单记录已具备
- 但正式收款仍未切到 `live`
- 结论：
  - 当前支付链路应视为“技术准备完成，可随时切生产”
  - 不是“已经正式对外收款”
