# 多语言同步固定流程

适用场景：英文主站发生文案、结构、下单路径、信任信息、FAQ、价格表达、CTA、表单字段等用户可见改动时。

目标：保证英文页与日文页在销售逻辑上保持一致，不出现“英文站已改、日文站还停留旧版本”的情况。

## 固定原则

1. 先改英文主链路，再同步日文。
2. 日文同步优先保证 `销售逻辑一致`，不要求逐字直译。
3. 任何影响转化的改动，都必须检查对应日文页是否需要同步。
4. 发布前必须同时检查英文页和日文页，不能只看英文页。

## 文件映射

- 英文首页：`index.html`
- 日文首页：`ja.html`
- 英文联系页：`contact.html`
- 日文联系页：`ja-contact.html`
- 英文支付页：`payment.html`
- 日文支付页：`ja-payment.html`
- 英文配送页：`shipping.html`
- 日文配送页：`ja-shipping.html`
- 英文保修页：`warranty.html`
- 日文保修页：`ja-warranty.html`
- 英文退货页：`returns.html`
- 日文退货页：`ja-returns.html`

## 执行步骤

### 1. 判断是否需要同步日文

出现以下任一情况，默认要同步日文页：

- 价格、套餐、容量、规格表达变化
- CTA 文案变化
- FAQ、信任文案、保修/退货/配送承诺变化
- 表单字段、下单步骤、支付路径变化
- 首页新增或删除重要销售模块

如果只是英文页的纯样式微调，且不影响信息结构，可不强制同步文案，但仍要确认日文页视觉没有受影响。

### 2. 先完成英文改动

先在英文页完成最终版本，不要英文和日文同时半成品推进。

要求：

- 英文页的文案逻辑先稳定
- CTA、价格表达、FAQ、模块顺序先定稿
- 不用在英文页还摇摆时提前翻译日文

### 3. 按映射同步日文页

同步时重点对齐这几类内容：

- 标题层级
- 价格解释逻辑
- 购买路径表达
- 关键卖点
- FAQ 里的风险说明
- CTA 去向

注意：

- 日文页可以更自然，但不能丢掉英文页新增的核心销售理由
- 如果英文页新增一个关键模块，日文页也要新增对应模块，而不是只改一句话

### 4. 做上线前检查

本地检查至少包含：

- 英文目标页内容正确
- 日文对应页内容正确
- 链接跳转未断
- 样式在桌面端和移动端不明显错位
- CTA 仍然指向正确页面

### 5. 发布后做双语验证

每次发布后，至少验证：

- 英文线上页是否出现本次新文案
- 日文线上页是否出现对应同步内容
- 缓存是否已经刷新到新版本

## 发布检查清单

- 英文页已完成最终文案
- 对应日文页已同步
- 关键 CTA 已复核
- FAQ / trust / pricing 逻辑双语一致
- 已完成线上英文页检查
- 已完成线上日文页检查

## 本项目当前默认做法

- 任何首页价格区、性能优势、下单流程、FAQ、信任信息改动
  都默认同步 `index.html` 和 `ja.html`
- 任何联系、支付、配送、保修、退货页改动
  都默认检查对应 `ja-*` 页面是否需要同步
- 如果本次改动没有同步日文，需要在提交说明或回复里明确写出原因
